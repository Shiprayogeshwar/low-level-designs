# Low Level Designs

This repository contains multiple Low-Level Design (LLD) problems.

Each problem is implemented in its own folder at the top level of this repository.

## Folder Structure

Each directory under this repository root represents one LLD problem.

Example:

```
low-level-designs/
  word_processor_using_flyweight/
```

## Convention

- One folder = one LLD problem
- Keep all code and supporting files for that problem inside its folder
- Use clear, problem-specific folder names (for example: `parking_lot`, `book_my_show`, `cache_with_lru`)


## Suggested Per-Problem Contents

Inside each problem folder, you can keep:

- Implementation files (Python classes/modules)
- A local `README.md` describing:
  - Problem statement
  - Design choices
  - Class diagram / object model (optional)
  - How to run and test

## Goal

Over time, this repository should become a collection of independently understandable LLD solutions, where each folder can be read and run on its own.
